# `.ply` Inspection Format (reference)

> Reference snippet for [`../05-sdf-pipeline.md`](../05-sdf-pipeline.md). Units are
> **meters**. Produced by CloudCropper's `PlyWriter` (`src/io/ply.cpp`). Loadable in
> RViz / Open3D / CloudCompare. Point-only PLY canNOT feed `GradientSDFField`
> directly — a mesh/SDF stage is required (doc 05 §2).

## Minimal (positions only)

```
ply
format binary_little_endian 1.0
comment Generated by CloudCropper
comment units=m frame=template          <-- PROPOSED (doc 05 §4.4), not emitted today
element vertex N
property float x
property float y
property float z
end_header
<binary body: N * (3 * float32)>
```

## Recommended (positions + normals)

```
ply
format binary_little_endian 1.0
comment Generated by CloudCropper
element vertex N
property float x
property float y
property float z
property float nx
property float ny
property float nz
end_header
<binary body: N * (6 * float32)>
```

## Mapping to CloudCropper

| PLY property      | CloudCropper                          | emitted today?               |
|-------------------|---------------------------------------|------------------------------|
| `x y z`           | `positions()` (always first)          | yes (`ply.cpp` ~L302)        |
| `nx ny nz`        | `"normal"` F32 arity-3 column         | yes, iff column present (`ply.cpp` ~L313) |
| `red green blue`  | `"rgb"` U8 arity-3/4 column           | yes, iff present             |
| `comment units=…` | `metadata()["units"]` / `["frame"]`   | **NO — proposed** (doc 05 §4.4) |

## Hard limitations (from `src/io/ply.cpp`)

- **Vertex element first, no list properties.** The reader rejects a non-`vertex`
  first element and rejects list props on vertex (`ply.cpp` ~L177, L180). The writer
  only ever emits a `vertex` element. ⇒ **no faces** — this is a point-only PLY.
- ASCII or `binary_little_endian` only (no big-endian).
- Metadata is not yet written (only a fixed `comment Generated by CloudCropper`).
