# Runtime deps for the rap worker + the (to-be-)vendored RAP core (PRBonn/RAP,
# rectified_point_flow package, built on Rectified Point Flow / NeurIPS 2025).
# Install into the interpreter named by `python:` in config/rap.yaml, e.g.:
#   python3 -m venv ~/venvs/rap && ~/venvs/rap/bin/pip install -r requirements.txt
#
# torch + numpy are required for the worker to come up `ready`; the rest are
# needed for actual RAP inference once the core is vendored.
#
# RECON (docs/design/_rap-upstream-notes.md): RAP pins a TIGHT version lattice
# (Python 3.10 + torch 2.5.1 + cu124 + pytorch3d 0.7.8 cu121 wheel + flash-attn
# cu12torch2.5 cp310). Deviating from any pin tends to break a wheel — this is a
# conda env, not a casual pip install. flash-attn is a HARD dependency (bare
# `import flash_attn`, no SDPA fallback) and its pinned 2.7.4 wheel does NOT
# support RTX 50xx/Blackwell (sm_120) — the single biggest deployment blocker.
